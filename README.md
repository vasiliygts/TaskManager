# Windows System Monitor — аналіз логів подій та менеджер процесів

Настільний застосунок на **Windows Forms** для моніторингу активних процесів та аналізу журналів подій Windows (Event Log). Програма поєднує дві основні функції:

- **Менеджер процесів** — перегляд, пошук та завершення процесів, моніторинг використання пам’яті.
- **Аналізатор логів подій Windows** — перегляд системних журналів (System, Application, Security тощо) з фільтрацією за типом та датою.

---

## Основний функціонал

### 1. Менеджер процесів

Форма **Process Manager** (Form1) реалізує моніторинг активних процесів у системі:

- Вивід списку запущених процесів у вигляді таблиці:
  - назва процесу;
  - **PID** (ідентифікатор процесу);
  - споживання пам’яті (MB);
  - час запуску;
  - заголовок вікна;
  - шлях до виконуваного файлу;
  - пріоритет процесу.
- **Пошук процесів**:
  - текстове поле для пошуку за частиною назви процесу;
  - відображаються лише ті рядки, що відповідають введеному фільтру.
- **Завершення процесу**:
  - вибір процесу в таблиці;
  - кнопка *«Завершити процес»* (kill) — спроба завершити вибраний процес за PID;
  - обробка винятків (системні/захищені процеси можуть не дозволяти завершення).
- **Автоматичне оновлення**:
  - таймер, що регулярно оновлює список процесів;
  - можливість змінювати інтервал оновлення (у секундах) під час роботи програми;
  - ручне оновлення кнопкою *«Оновити»*.
- **Збереження контексту**:
  - під час оновлення зберігаються:
    - вибраний процес (за PID);
    - поточне сортування таблиці;
    - позиція прокрутки.
- **Підсумкова інформація**:
  - загальна кількість процесів;
  - сумарне використання оперативної пам’яті (MB).

---

### 2. Аналізатор логів подій Windows

Форма **Event Log Viewer** (*EventLogForm*) забезпечує роботу з журналами подій Windows:

- **Вибір журналу**:
  - комбобокс зі списком доступних системних журналів:
    - System;
    - Application;
    - Security;
    - інші журнали, доступні у системі.
- **Фільтрація за датою**:
  - два поля вибору дати: *«З»* (dtpFrom) та *«По»* (dtpTo);
  - завантажуються тільки події з обраного діапазону.
- **Перевірка прав доступу**:
  - при спробі відкриття журналу **Security** програма перевіряє, чи запущена з правами адміністратора;
  - якщо прав недостатньо — відображається попередження та завантаження подій не виконується.
- **Вивід подій у таблиці**:
  - № (порядковий номер запису);
  - час події (TimeGenerated);
  - тип події (Information, Warning, Error тощо);
  - джерело (Source);
  - текст повідомлення (Message), за потреби обрізаний до ~200 символів із трикрапкою.
- **Зручність перегляду**:
  - автоматичне налаштування ширини колонок (короткі поля — авто по вмісту, повідомлення — займає решту ширини);
  - таблиця тільки для читання, без можливості редагування;
  - вимкнений лівий заголовок рядків (чистіший вигляд таблиці).

## Технології

- **Мова програмування:** C#
- **Платформа:** .NET Framework (Windows Forms App)
- **GUI:** Windows Forms (**System.Windows.Forms**)
- **Робота з процесами та журналами:**
  - `System.Diagnostics.Process` — отримання інформації про процеси та їх завершення;
  - `System.Diagnostics.EventLog`, `EventLogEntry` — читання подій із системних журналів;
  - `System.Security.Principal` — перевірка прав адміністратора.
- **Обробка колекцій та фільтрація:**
  - LINQ (`System.Linq`) — фільтрація подій за діапазоном дат, обробка колекцій.
- **Розробка:**
  - Microsoft Visual Studio (Windows Forms Designer, відлагодження, запуск).

## Структура проєкту

(Назви файлів/класів можна скоригувати під фактичну структуру ріпозиторію.)

- `Program.cs`  
  Точка входу в застосунок, конфігурація запуску головної форми.

- `MainForm.cs` (якщо використовується)  
  Головне вікно / меню навігації:
  - кнопка відкриття менеджера процесів;
  - кнопка відкриття аналізатора логів подій.

- `Form1.cs`  
  Модуль **менеджера процесів**:
  - ініціалізація `DataGridView` для процесів;
  - метод `LoadProcesses()` — завантаження та оновлення списку процесів;
  - пошук, завершення процесів, зміна інтервалу оновлення;
  - робота з таймером автоматичного оновлення.

- `EventLogForm.cs`  
  Модуль **аналізатора подій Windows**:
  - завантаження доступних журналів (`EventLog.GetEventLogs()`);
  - фільтрація подій за датою;
  - перевірка прав доступу до журналу Security;
  - заповнення таблиці подій.

- `*.Designer.cs`  
  Автоматично згенеровані файли дизайнера форм (розташування контролів, властивості елементів інтерфейсу).
